Install

  • Install

    • Unzipping packages

    Depending on the services you purchased, you have been provided with various .zip archives. Extract them in the same destination folder, in order to access all the code, tools and resources needed for building your own handwriting application. The file hierarchy should look like this:

    Description

    doc

    Contains the HTML documentation files. The file index.html displays the main page.

    edk

    Contains the Engine Development Kit and handwriting programming elements for each supported programming language API, including code samples.

    engine

    Contains dll libraries and the security certificate.

    rdk

    Contains the Resource Development Kit, i.e. tools and examples for creating custom resources.

    resources

    Contains resource files with the .res extension. They are binary assets used at run time by the different MyScript technologies to achieve the various recognition tasks.

    tools

    Contains useful programmer tools including InkTool for ink testing.

    Setting the certificate

    A valid certificate is mandatory to use the MyScript SDK. This is a security measure that uniquely identifies you as a legitimate customer of MyScript technologies. The certificate helps us trace customers identity and the products that have been purchased.

    We generate a unique certificate for each MyScript product you purchase. Your certificate allows you to activate that product and it will keep working even if your valid support period has expired.

    For each new product you buy, your certificate will be updated.

    Generating the certificate code

    The certificate is delivered in the [your_login].vo.zip package. When you unzip this package, the certificate is automatically put in the appropriate locations. This makes sure that the certificate is instantly working within an application.

    The delivered files containing the certificate and their locations depend on your programming language:

    Programming language Files Location
    C MyCertificate.bin, MyCertificate.c, MyCertificate.h engine/certificates, edk/c/examples/certificates
    Java MyCertificate.bin, MyCertificate.java engine/certificates, edk/java/examples/certificates
    .NET MyCertificate.bin, MyCertificate.cs engine/certificates, edk/dotNET/examples/cs/certificates

    Running a recognition

    No MyScript technology can be used without a genuine certificate.

    See the Tutorial section to learn how to achieve this.

    Code samples

    Code samples are located in the edk/[programming language]/examples/ directory.
