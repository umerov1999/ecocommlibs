MyScript Ink

  • SDK
  • Ink

    • About MyScript Ink

    MyScript Ink is designed for building applications that enable creation and edition of digital ink.

    In brief, MyScript Ink supplies:

    Item Description Location
    API for C, Java and .NET Ink API edk/your_programming_language/
    MyScriptInk The specific library engine/
    CreateInk, EditInk, UsingTags These source code samples show you how to create and edit ink and how to use ink tags edk/your_programming_language/example/ink/
    MyScript Ink SDK provides tools for modeling, editing, managing digital ink. It does not provide ink recognition service.

    Application blueprint

    MyScript Ink offers various ways to manipulate digital ink. Here is a possible workflow:

    1. Create an ink object. MyScript Ink models the input digital ink 1 through an ink object. This object stores different stroke coordinates and other features.

    2. From this ink object, you can manipulate, edit, transform, tag any portions of ink using the available API methods.

    3. You can also be notified of any change in the digital ink through callback mechanisms implemented in this SDK.

    The following section describes the main objects used in MyScript Ink environment.

    Environment

    The main objects of MyScript Ink environment are presented hereby.

    Ink

    An ink object models the digital ink. It contains several objects describing ink features, among which are:

    • a set of inkStroke objects and,

    • a set of tags.

    InkStrokeBuilder

    An inkStrokeBuilder object allows constructing a new inkStroke.

    InkStroke

    An inkStroke object models one stroke of the ink, that is any ink mark, geometric primitive or glyph (digital character). This object contains a set of values that describes different channels. A channel can represent coordinates of the ink mark (e.g. X and Y channels), timestamps, pressure, etc. There is one array of values per channel. Each inkStroke is associated with one inkStrokeFormat.

    InkStrokeFormat

    The inkStrokeFormat object holds information regarding one inkStroke. For instance, it contains data such as :

    • the number of channels,

    • the name of these channels and,

    • the value type of the channels (float, etc.) of this inkStroke.

    For example, a typical inkStrokeFormat can indicates that the inkStroke it is linked to contains 2 channels named X and Y whose value types are float.

    Tag

    The tag is a concept that indicates a piece of information on a given portion of ink. It is not an object that you can directly instantiate but you can use it via API methods.

    There are various cases tags are useful for. Ink may be tagged by MyScript operations to associate a portion of ink with recognition results, for instance. Ink can also be tagged by the user, for example to associate a portion of ink with layout information.

    Creating an ink object

    Creating an ink object requires to achieve the following:

    1. Create an empty ink object with the createInk method.

    2. Create an inkStrokeBuilder to enable the construction of a new stroke using the createStrokeBuilder method. Provide this method with the description of the stroke format using an inkStrokeFormat object.

    3. Add each sample point forming this stroke to the inkStrokeBuilder through the addSample method.

    4. When you have added all the sample points forming the stroke to the inkStrokeBuilder, you can finally create this stroke using the createStroke method. This creates the inkStroke object.

    5. Add the created inkStroke to the ink object with the addStroke method.

    Editing

    Once you have created the ink object with inkStroke elements, you can perform various editing tasks, such as erasing, chopping, pasting, applying geometrical transforms, etc.

    To apply an edition operation, you will need to use an inkIntervals object. This object defines the portion of ink to consider. In the following, we introduce the inkSelection object, which is a particular inkIntervals object, because it is more likely to be used in most of the edition use cases. However, you can need to use another inkIntervals object that is the inkSegment in some cases. Check the section Intervals to learn more on inkIntervals.

    Here are the main steps to do so:

    1. Create an inkSelection using the createInkSelection method. An inkSelection defines a portion of ink. You can modify the extent of the inkSelection whenever necessary.

    2. Set the selection with a portion of ink using a selection function. For instance, use selectAll to assign the entire ink to the inkSelection, or selectRectangle to assign only the ink contained inside a specified rectangular shape.

      At creation, an inkSelection is empty. To modify it with a selectionModifier, see the operations below.
      An inkSelection is associated with one given ink object only. It can not contain items from another ink object.

    3. Apply the editing operation using one of the available editing methods, such as erase to delete all the ink referred to by the inkSelection object, for instance.

    Selection methods

    There are different methods for you to set inkSelection objects:

    Operation Description
    selectAll selects the entire ink
    selectNone empties the selection
    selectRectangle, selectCircle, selectPolygon selects the ink contained inside a geometric shape
    combine selects another selection
    selectStroke selects a given stroke
    hitStroke selects the closest stroke from a given point

    For each of these methods, except selectAll and selectNone, a modifier must be set. A modifier indicates how the current selection must be combined with the operation selection. In the API, we refer to a modifier as a selectionModifier object. The possible values of the selectionModifier are:

    • SET: the new selection will contain only the operation selection,

    • UNION: the new selection will contain the union of current selection and operation selection,

    • INTERSECTION: the new selection will contain only the intersection between current selection and operation selection,

    • DIFFERENCE: the new selection will contain the current selection from which the operation selection has been removed.

    The figure below illustrates the steps for setting a selection.

    The content of an inkSelection is automatically updated to match the modifications that are made to the ink, namely erase, transform and chop operations. For example, if an ink selection contains a stroke that gets removed from the ink, that stroke is automatically removed from that inkSelection.
    You can ensure that only entire strokes are selected using the method adjustToStrokeBoundaries.
    You can use ink tags to adjust the selection to the ink tagged with a particular name through the adjustToTagBoundaries method.

    Edition methods

    The different methods enabling edition of ink are described hereby. All of the edition methods require to be indicated the ink portion to modify, i.e. ink intervals. This can be done using an inkSelection or an inkSegment. Check the section Intervals to learn more on ink intervals.

    Chop

    The chop method is used to cut strokes in 2 (or more) pieces. Chopping is performed at the boundaries of the input ink intervals.

    The points on chop boundaries are not included in the resulting strokes.

    Erase

    The erase method is used to erase ink intervals. If some tags become empty following this operation, they are automatically deleted.

    Paste

    The paste method is used to copy data from an ink object and paste it into another ink object.

    Data to paste may be an ink object or an inkInterval object and it may point to the same ink as the ink to modify. The paste operation will copy the strokes and the tags within the data to paste.

    Transform

    The transform method is used to transform portions of ink described through inkIntervalsobjects.

    The transformation is described by a matrix, which is a set of 6 float values {a,b,c,d,e,f}. The transformation of a point (x,y) is computed as follows:

    x’ = a*x + b*y + c
    y’ = d*x + e*y + f

    MyScriptInk provides you with helpers that apply the following operations to a transformation matrix:

    These helpers also enable basic operations on the matrices:

    • setIdentity: resets the matrix to identity,

    • multiply: combines two transformations,

    • inverse: computes the inverse transformation.

    If you perform a transformation operation on a partially selected stroke, this stroke will be chopped, because part of this stroke is transformed but the rest remains unchanged.

    Tagging

    You can add pieces of information to ink portions thanks to the concept of the tags.

    Tagging a portion of ink requires various steps:

    1. Create an ink portion. Portions of ink are defined through inkIntervals objects. Those represent the union of a collection of strokes or portions of strokes. An inkSelection object is a particular type of inkIntervals. So, you can create an inkSelection from an existing ink object to create a tag. Then you set the selection using a selection method.

    2. Create and add a new tag to the ink using the addTag method. Provide this method with the details on this tag (its name for instance, through a string, and an optional JSON data object). At this point, MyScript Ink automatically assigns the new tag with a unique tag ID.

    3. You can access the list of tags using the methods tagLookup and tagLookupCount.

    You can create an empty tag, i.e. an ink tag that is not associated with any inkIntervals. That ink tag can then be modified to be associated with ink.
    You can create several ink tags with the same name and on the same ink intervals, but their tag ID will always differ. The tag ID is generated by MyScript Ink so that it is a unique identifier.
    The edition of existing ink automatically updates the ink tag set. In particular, if the ink portion associated with a given ink tag is entirely erased, then that ink tag is removed.

    Ink change notifications

    To be automatically notified when a change occurs, you can implement a listener and register it to the ink object or the inkSelection object that you want to monitor.

    MyScript Ink offers several ways to monitor the changes that happen on ink and on ink selections.

    Getting notified on ink changes

    The steps required to be notified in case of ink changes are:

    1. Create an inkUpdateListener object to enable your application to receive updates on ink changes. To do so, you need to create a UserObject and to register it as a VO_IInkUpdateListener through the method registerInterface.

    2. Attach the inkUpdateListener to the ink object through the method addListener.

    3. The digital ink has changed from the user side.

    4. The inkUpdateListener then notifies your application that a changed occurred through the callback method onUpdate. You can implement this callback method according to your needs. This method provides you with the following pieces of information:

      • the extent of the modifications,

      • a detailed list of information on the ink tags that have been modified. This contains for each ink tag:

        • the tag ID,

        • an indication whether that ink tag was added, updated or removed,

        • an indication of whether the change was due to a transform operation only or to another operation (for example strokes added or removed).

    Getting notified on ink selection changes

    At this stage, you have already created an InkSelection object.The steps required to be notified in case of ink selection changes are:

    1. Create an inkSelectionChangeListener object to enable your application to receive updates on ink changes. To do so, you need to create a UserObject and to register it as a VO_ISelectionChangeListener through the method registerInterface.

    2. Attach the inkSelectionChangeListener to the inkSelection object through the method addChangeListener.

    3. The digital ink has changed from the user side and it matches the ink selection defined by the inkSelection object.

    4. The inkSelectionChangeListener then notifies your application that a change occurred through the callback method changed. You can implement this callback method according to your needs. The notification will indicate if the change was due to a transform operation only or to another operation (for example strokes being added or removed) using flags.

    Advanced use

    Stroke types

    The inkStroke object has been defined in a previous section. As explained, an inkStroke models any kind of strokes. To represent the different kinds of strokes more specifically, there are other objects that derive from the basic inkStroke object:

    • the arcPrimitive,

    • the linePrimitive,

    • and the glyph.

    LinePrimitive

    The linePrimitive is one of the two primitives supported by MyScript Ink. It represents the straight line and it is described by its 2 points.

    As a primitive object, the linePrimitive can be decorated. In that case its first and last points are associated with a flag indicating their decoration, which can be:

    • None

    • Arrow Head

    ArcPrimitive

    The arcPrimitive is one of the two primitives supported by MyScript Ink. It represents the elliptic arc and it is described by either an end point parameterization or a center parameterization.

    The following figures depict an overview of what arcPrimitive objects parameterizations describe:

    Like the linePrimitive, the arcPrimitive can also be decorated.

    Glyph

    A glyph represents a character, it is described by:

    • a bounding box, that indicates the position and size of the character,

    • a list of possible character alternates, with the probability of their occurrence.

    You can use glyph objects for typesetting a recognized ink sample:

    You can also use glyph objects for importing digital characters.

    glyph objects do not contain sample points, so you can not access a glyph set of points, unlike the generic inkStroke objects and the primitives.
    In case you apply a selection on a glyph, this object cannot be partially selected. If a selection overlaps a glyph, it is entirely selected.
    You can chop inkStroke objects and primitives, but you cannot chop a glyph.
    The extent of a rotated glyph is an axis aligned bounding box. The rotation information is kept internally.

    You can create glyph objects by specifying glyph reference line positions. The figure below shows different letters with their bounding boxes and the reference lines.

    You can set the glyph lines in the API through the glyphLines object. However, you can not provide the baseline and the x-height values directly, but rather as ratios. Check the API.

    Intervals

    As introduced in previous sections, inkIntervals object refer to union of collections of strokes or portions of strokes. To be more precise, an inkIntervals object is the union of inkInterval objects.

    An inkInterval describes the collection of strokes contained between 2 cursors, modeled as inkCursor objects. It may contain partial strokes.

    When creating an inkSelection, you can select inkIntervals or inkInterval objects:

    Methods Description
    selectInterval selects an interval
    selectIntervals selects an ink segment or another ink selection

    There are two types derived from the inkIntervals object:

    • the inkSelection, which can be used to apply change on the portions of ink it refers.

    • and the inkSegment, which can also be used to apply changes on ink portions but you can not change an inkSegment values it once it is instantiated.

    InkCursor

    An inkCursor represents a position in the ink. It is used to describe a inkInterval object.

    It is described by the following parameters:

    • the index of a stroke (ink stroke, glyph or primitive) in the raw strokes,

    • a point index within that stroke. We use decimals to indicate a position between two points of a stroke. For example, a value of 1.2 indicates a position between point 1 and 2, at 20% of the distance between point 1 and 2.

    • a Boolean: included. It is used to indicate whether the position is contained in an interval or not.

    InkSegment

    An inkSegment is an inkIntervals that is immutable. Unlike an inkSelection, you can not change the content of an inkSegment once it is created. You can use it to indicate the ink portion to consider when applying an edition operation or when tagging ink.

    You can create an inkSegment from an inkSelection. In this case, it represents the content of that selection at that point in time. If that selection is modified, the inkSegment remains unchanged.

    An inkSegment is associated with one given ink only, it cannot contain items from another ink.

    Raw strokes and active strokes

    Consider an ink where 10 strokes have been added, and no other modification has been done.

    If strokes 0 to 8 are erased, you want to keep the information of the ink as it was at the initial point, e. g. before erasing. In a similar manner, you want to know the strokes that remain currently. For this reason, MyScript Ink introduces the concept of raw strokes and active strokes.

    Raw strokes

    Raw strokes represent the strokes as they were when they were added to the ink, even if they have been erased or transformed afterward. Raw strokes are used to describe ink intervals, because it ensures a stable referencing of the strokes.

    In the previous example, raw strokes remain unchanged. Raw stroke 9 will always be raw stroke 9, even if previous strokes have been erased or transformed.

    Active strokes

    Active strokes represent the strokes, including the modifications that were applied to them, whether they have been transformed, partially deleted, chopped, etc. The samples of the active strokes can be retrieved from an ink, ink segments or ink selections.

    In the previous example, active strokes contain only one stroke. The remaining active stroke will be referenced as:

    • stroke 9 in the raw strokes,

    • stroke 0 in the active strokes.

    Advanced use of tags

    Tag query

    MyScript Ink allows performing a query on ink tags. They can be queried based on one or more of the following parameters:

    • name of the tag,

    • tag ID of the tag,

    • ink intervals, i.e. the query looks for tags whose ink segment either includes, matches or intersects the ink intervals provided as query input.

    The outcome of the query is an iterator that allows user accessing the ink tags parameters. That iterator points to an ink tag on which you can perform operations:

    • remove the ink tag (this will remove the tag, not the ink it references),

    • change the data object associated with the ink tag.

    Tag edition

    You can update an ink tag to change the ink segment it references.

    You can also select an ink tag from its tag ID with an ink selection, and then you can perform any ink edition operation on the portion of ink referenced by this ink tag.

    Other use cases with tags

    When adding a stroke to an ink, you can provide a list of ink tag IDs, to automatically tag this stroke with existing tags.

    You can also paste strokes from another ink. In this case, the tags associated with the pasted strokes are copied into the new ink. The tag IDs of the copied tags are automatically modified by MyScript Ink, to ensure their uniqueness.

    Jink file format

    Jink stands for JSON-based ink format. It is a file format meant to store not only ink strokes data, but also glyphs and geometric primitives, together with tagging information.

    You can save data to Jink format.

    Below you will find an overview of the items that can be found in a Jink file.

    Stroke Format

    The stroke format describes how ink strokes are stored in the file.

    A stroke format is a list of channels, which are defined by their name and type. For example a simple stroke format would consist in a X and a Y channel, that both have a float32 type.

    In addition to the channel list, stroke format may indicate the sampling rate and whether this rate is uniform or not.

    In a Jink file, you can define several stroke formats. These formats are stored in a formats table, and when defining strokes, the associated format is referenced by its index in that table.

    Here is an example:

    "formats": [
 {
  "channels": [
   {
    "name": "X",
    "type": "float32"
   }, 
   {
    "name": "Y",
    "type": "float32"
   }, 
   {
    "name": "T",
    "type": "int32"
   }
  ]
 },
 {
  "channels": [
   {
    "name": "X",
    "type": "float32"
   }, 
   {
    "name": "Y",
    "type": "float32"
   }
  ]
 }
]

    Ink Strokes

    Ink strokes are listed in a strokes array, together with primitives and glyphs.

    A stroke is described as arrays of values that match the channels described in one of the Jink file ink stroke format. That format is referenced under the format member, by its index in the stroke format table.

    Here is an example where a stroke contains 8 points, and follows the stroke format 1 in the stroke format example.

    {
 "format": 1,
 "timestamp": "2013-09-02 15:09:31.187335",
 "X": [ 257, 257, 327, 327, 257, 327, 327, 257 ],
 "Y": [ 312, 372, 372, 312, 312, 372, 312, 372 ]
}

    Primitives

    Primitives are listed in a strokes, array together with ink strokes and glyphs.

    Depending on whether a primitive is a line or an elliptic arc, it is described by the corresponding geometric information (e.g. 2 points in the case of a line).

    If a primitive is decorated, then the decoration member of the primitive description indicates whether the first point and/or last point are decorated.

    Here is an example where primitives describe a line that ends with an arrow head and an elliptic arc:

    {
 "item-type": "line",
 "timestamp": "2013-09-02 15:09:31.187335",
 "decoration": [ 0, 1 ],
 "p1": [ 100, 120 ],
 "p2": [ 1100.2, 1120.3]
},
{
 "item-type": "arc",
 "timestamp": "2013-09-02 15:09:31.187335",
 "cx": 200,
 "cy": 150,
 "rx": 1.02,
 "ry": 1.5,
 "phi": 1.6,
 "startAngle": 1,
 "sweepAngle": 6
}

    Glyphs

    Glyphs are listed in a strokes array, together with ink strokes and primitives.

    A glyph is described as a bounding box and a list of possible alternates (characters associated with their probability of occurrence).

    Here is an example where the glyph contains a c or a C:

    {
 "item-type": "glyph",
 "timestamp": "1970-01-01 00:00:00.000000",
 "extent": [ 300, 3650, 400, 3750 ],
 "alternates": [
  {
   "label" : "c",
   "probability" : 0.6
  },
  {
   "label" : "C",
   "probability" : 0.4
  }
 ]
},

    Ink cursors

    An ink cursor describes a position in the ink. It associates the index of an ink item (stroke, glyph or primitive) in the raw strokes, with a position within that stroke.

    In the example below, ink cursor points to point 2 in ink item 0. In the second example, ink cursor points between point 2 and 3 of ink item 0, at 0.125 times the distance between those 2 points (so closer from point 2 than from point 3).

    0:2
0:2.1255555

    Ink intervals

    An ink interval is described by a start cursor and an end cursor, contained between brackets that may be open or closed, depending on whether the first (or last) point is included in the interval.

    In the first example below, you can see that the ink interval ranges from the first ink item (included entirely), to point 6 in ink item 20, included. In the second example, the ink interval ranges from right after the first point of the first ink item, to right before point 6 of ink item 20.

    [0:0,20:6]
]0:0,20:6[

    Ink intervals can be combined with a U:

    [0:0,20:6]U]21:6.765,21:13]U[22:0,40:12.5[

    Raw strokes, active strokes and transforms

    The strokes, glyphs and primitives listed in the strokes array represent the raw strokes of the ink, i.e. the ink items as they were when they were added to the ink, even if they have been erased or transformed afterwards.

    Modifications that are applied on the raw strokes are described separately, as follows:

    • the ink intervals of the items that have not been erased are described in active-strokes.

    • The transformation matrices, and the ink intervals they apply to, are described in transforms.

    Here is an example:

    "active-strokes": "[0:0,20:6]U]21:6.765,21:13]",
"transforms": [
 {"segment": "[0:0,13:11]", "transform": "matrix(1, 0, 258.908, 0, 1, 1.318)"},
 {"segment": "[14:0,20:6]", "transform": "matrix(2.282, -2, 510.25, 0.009, 3.3, -727.985)"},
],

    Tags

    Ink tags are described as follows:

    • the tag name,

    • the tagged ink segment, expressed as an ink interval,

    • the tag id,

    • an optional data member, that may contain a string or a JSON object.

    In addition the next tag id to be used as new-tag-id. This value is used to ensure the same tag id is not used twice.

    Here is an example:

    "tags": [
 { "name": "TextStroke", "segment": "[14:0,20:6]U]21:6.765,21:13]", "id": 0 },
 { "name": "Drawing", "segment": "[0:0,13:11]", "id": 2 },
 { "name": "Word", "segment": "[14:0,17:8]", "data": "hello", "id": 3 },
],
"new-tag-id": 4

    1. As defined in Overview, digital ink refers to the series of strokes that define the pens trajectory. The word ink in MyScript Ink is used to refer to the API object that models this digital ink.
